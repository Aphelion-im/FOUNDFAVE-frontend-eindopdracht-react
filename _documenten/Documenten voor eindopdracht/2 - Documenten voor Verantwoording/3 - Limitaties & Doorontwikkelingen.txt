## Toelichting op de limitaties
1) Marvel API
Ik had gehoopt dat de Marvel API meer informatie zou verstrekken over de Marvel films en series, zoals op de Marvel.com site zelf.
In plaats daarvan kun je alleen beknopte informatie van de karakters uit de comics opvragen. Ook enkele resources werken niet, terwijl deze wel zijn op te vragen via de endpoints en geven een 404 error. 
Ik kwam er later achter dat deze API te beperkt is voor mijn plannen. 

Ik ben toch met deze API doorgegaan omdat de API al was doorgelicht door NOVI Hogeschool en ik had al toestemming om door te gaan met dit idee.
 
Het zou een risico zijn geweest, zoals NOVI al aan gaf in het lesmateriaal, om een eigen API op te zoeken.

Daarbij is deze API goed genoeg voor de 5 functies van de eindopdracht.

2) Gebruikersaccount verwijderen. 
Naar ik weet is er geen endpoint of optie aanwezig in de NOVI-backend om een gebruiker te verwijderen.
Dit zal ik in de toekomst zelf ontwikkelen met bijvoorbeeld Firebase.

3) Favorieten categorie aanmaken, verwijderen en bewerken. (Zoals op Bol.com)
In de huidige app kun je alleen in een algemene lijst favorieten toevoegen en verwijderen.
De toekomstige app zal de mogelijkheid hebben om een categorie aan te maken en favorieten opnieuw in te delen in de gewenste categorie.

De huidige app maakt gebruik van localStorage om de favorieten in op te slaan. De toekomstige app zal gebruik maken van een database.

4) De WCAG-contrast richtlijnen heb ik achterwege gelaten. Tijdens het ontwerpen heb ik er losjes rekening mee gehouden, maar heb niet alle achtergrond-en-lettercombinaties doorgemeten of de contrastratio voldoende is.
Aangezien NOVI het grafische ontwerp niet het belangrijkst onderdeel vindt voor de eindopdracht, laat ik dat achterwege. 
Hetzelfde geldt voor het Aria-label voor slechtzienden.

In de toekomst wil ik ARIA gaan inbouwen in de formulieren. 
ARIA is bedoeld om interface-elementen te kunnen voorzien van context en werking om toegankelijkheid te verbeteren.
ARIA maakt, onder andere, de formulieren toegankelijker voor mensen met screenreaders.

5) Omdat de huidige app helaas geen toegang heeft tot een database zijn er een aantal functies niet mogelijk.
- Een review plaatsen.
- Een rating geven.
- Categorie aanmaken, verwijderen en bewerken.
- Gebruikersdata opslaan.
- E-mail alarm krijgen.
- E-mail tijdens een wachtwoord reset.
- E-mail sturen na registratie.
- Een account verwijderen.
- Profiel bekijken en aanpassen.
